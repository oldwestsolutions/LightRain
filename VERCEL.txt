LightRain on Vercel (one Git repo, three subdomains)

1) Create THREE Vercel projects, all linked to this repo.

2) For each project, set Root Directory (Settings → General → Root Directory):
   - Marketing:  apps/web
   - Wallet:     apps/wallet
   - Admin:      apps/admin

   Leave Install Command / Build Command empty so each app’s vercel.json runs:
   "cd ../.. && npm install" and "npm run build" (cwd is already apps/<app>, so .next is at path0).

3) If you MUST leave Root Directory as "." (repo root) on a project:
   - Add env var LIGHT_RAIN_VERCEL_APP = web | wallet | admin (match the app).
   - Install: npm install
   - Build: npm run build
   - Root vercel.json sets framework nextjs and outputDirectory null (overrides a dashboard
     Output Directory of "public"). After build, the script copies that app's .next,
     next.config.ts, and public to the repo root (symlinks are not reliable on Vercel).
     If LIGHT_RAIN_VERCEL_APP is omitted, the build defaults to web and warns (set the var
     for wallet/admin root deploys).
   Prefer Root Directory = apps/* instead (avoids brittle repo-root tracing).

   If ".next not found at path0" persists on a repo-root project, in Vercel → Settings →
   Build & Deployment set Output Directory to apps/web/.next (or wallet/admin) for that
   project instead of relying on the post-build copy.

4) Domains (Settings → Domains):
   - web    → lightra.in / www
   - wallet → wallet.lightra.in
   - admin  → admin.lightra.in
